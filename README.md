# EQEmu GM Dashboard

A real-time GM activity dashboard for [EQEmu](https://www.eqemu.net/) servers.
Tracks GM commands, item grants, kills, player deaths, bot activity, level progression,
character status, and Spire audit logs through a dark-themed web interface.

![Dashboard preview](docs/preview.png)

---

## Features

- **New Commands** — live feed of GM commands since last visit, highlighted in real-time
- **All Commands** — full searchable command history with item and spell popups
- **Kills & Items** — itemized log of everything given, with rarity colors and item lookups
- **Trends** — activity heatmap by day/hour and session timeline
- **Characters** — level history and bot inventory viewer
- **Bots** — bot roster with class/race breakdown
- **Spire Log** — Spire audit trail (what GMs did in the UI)
- **Summary** — highlights: top kills, top items given, top recipients, peak hours

---

## Requirements

- Docker and Docker Compose
- An EQEmu server running [akk-stack](https://github.com/Akkadius/eqemu-akk-stack)
- [Spire](https://github.com/Akkadius/spire) (optional — enables item/spell lookups)
- The `gm_export.sh` daemon runs on the host or in a scheduled job

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/eqemu-gm-dashboard.git
cd eqemu-gm-dashboard
```

### 2. Configure

```bash
cp .env.example .env
nano .env   # or vi, or your editor of choice
```

Minimum required settings in `.env`:

```env
DB_HOST=127.0.0.1          # IP of your MariaDB host
DB_PASS=your_password_here  # MariaDB root password
SPIRE_URL=http://127.0.0.1:3000
TZ=America/Chicago          # Your timezone
REPORTS_DIR=/tmp/eqemu-reports
SCRIPTS_DIR=/path/to/eqemu-gm-dashboard
STACK_DIR=/path/to/akk-stack   # Directory with akk-stack .env
```

### 3. Build and start

```bash
docker compose up -d --build
```

The dashboard is now available at `http://YOUR_SERVER_IP:8765`.

### 4. Start the export daemon

The export daemon runs `gm_export.sh` on your host machine every 60 seconds,
querying the EQEmu database and regenerating the report.

**Option A — run manually (test first):**
```bash
bash gm_export.sh
```

**Option B — run on startup (Linux systemd):**
```bash
sudo cp gm_export.sh /usr/local/bin/eqemu-gm-export
sudo chmod +x /usr/local/bin/eqemu-gm-export

# Create a systemd service
sudo tee /etc/systemd/system/eqemu-gm-dashboard.service << 'SERVICE'
[Unit]
Description=EQEmu GM Dashboard Export Daemon
After=docker.service

[Service]
Type=simple
EnvironmentFile=/path/to/eqemu-gm-dashboard/.env
ExecStart=/usr/local/bin/eqemu-gm-export
Restart=always

[Install]
WantedBy=multi-user.target
SERVICE

sudo systemctl enable --now eqemu-gm-dashboard
```

**Option C — Unraid User Scripts:**

In Unraid, add `gm_export.sh` as a User Script set to run
**"At Startup of Array"** with no cron schedule.
Make sure the `.env` file is in the same directory as the script,
or set all variables directly in the Unraid script environment.

## Configuration Reference

All settings live in `.env`. See `.env.example` for the full list.

| Variable | Default | Description |
|---|---|---|
| `TZ` | `UTC` | Timezone for timestamps (tz database name) |
| `DASHBOARD_PORT` | `8765` | Port to access the dashboard |
| `DB_HOST` | — | MariaDB host IP or hostname |
| `DB_PORT` | `3306` | MariaDB port |
| `DB_USER` | `root` | MariaDB username |
| `DB_PASS` | — | MariaDB password (**required**) |
| `DB_NAME` | `peq` | EQEmu database name |
| `SPIRE_URL` | `http://localhost:3000` | Full URL to your Spire server |
| `REPORTS_DIR` | `/tmp/eqemu-reports` | Where reports are written on the host |
| `SCRIPTS_DIR` | Script directory | Directory containing `gm_excel.py` |
| `STACK_DIR` | `/opt/eqemu/server` | Path to your akk-stack directory |
| `INTERVAL` | `60` | Seconds between report refreshes |
| `MARIADB_CONTAINER` | `mariadb` | Docker container name for MariaDB |
| `WORKER_CONTAINER` | `eqemu-gm-worker` | Docker container name for the worker |
| `MARIADB_ROOT_PASSWORD` | — | Set directly instead of reading from akk-stack `.env` |

## Spire Token (for item/spell lookups)

The item and spell lookup cards fetch data through Spire's API.
To enable them, generate a Spire API token and save it:

```bash
# Inside Spire, go to Settings → API Tokens → Generate
echo "your_token_here" > /path/to/eqemu-gm-dashboard/spire_token.txt
```

Then set `REPORTS_DIR` to the same directory so the server can find it.

## Architecture

```
Host machine
├── gm_export.sh (daemon, runs every 60s)
│   ├── Queries MariaDB via docker compose exec
│   ├── Writes TSVs to /tmp
│   └── Calls: docker exec eqemu-gm-worker python /scripts/gm_excel.py
│
Docker containers
├── eqemu-gm-dashboard  (reports_server.py, port 8765)
│   ├── Serves index.html (generated by gm_excel.py)
│   ├── Proxies Spire API requests
│   └── Exposes /spell-search, /spire-user-log, /inventory/* endpoints
│
└── eqemu-gm-worker  (idle, exec'd by gm_export.sh)
    ├── Mounts /tmp as /data  (reads TSV input files)
    ├── Mounts SCRIPTS_DIR as /scripts  (reads gm_excel.py)
    └── Mounts REPORTS_DIR as /output  (writes index.html + .xlsx)
```

## Troubleshooting

**Dashboard shows no data / blank sections**
- Confirm `gm_export.sh` is running: `ps aux | grep gm_export`
- Check the worker container is up: `docker ps | grep eqemu-gm-worker`
- Verify TSV files are being written: `ls -la /tmp/eqemu_gm_*.tsv`

**"DB not configured" error**
- Set `DB_HOST` and `DB_PASS` in your `.env` file
- Confirm MariaDB is reachable from the host: `mysql -h $DB_HOST -u root -p$DB_PASS peq -e "SELECT 1;"`

**Spell/item lookups not working**
- Confirm Spire is running and `SPIRE_URL` is correct
- Check your Spire token file exists and is readable

**Level cap issues / wrong XP rate**
- See the [rule_values tuning guide](docs/rules.md) (coming soon)
